II. Code Readability (20% of your project grade)

To receive the full readability marks, your code must follow the following guideline:

  • All variables (fields, parameters, local variables) must be given appropriate and descriptive names.
  • All variable and method names must start with a lowercase letter. All class names must start with an uppercase letter.
  • The class body should be organized so that all the fields are at the top of the file, the constructors are next, the non-static methods next, and the static methods at the bottom with the main method last.
  • There should not be two statements on the same line.
  • All code must be properly indented (see Appendix F of the Lewis book for an example of good style). The amount of indentation is up to you, but it should be at least 2 spaces, and it must be used consistently throughout the code.
  • You must be consistent in your use of {, }. The closing } must be on its own line and indented the same amount as the line containing the opening {.
  • There must be an empty line between each method.
  • There must be a space separating each operator from its operands as well as a space after each comma.
  • There must be a comment at the top of the file that is in proper JavaDoc format and includes both your name and a description of what the class represents. The comment should include tags for the author. (See Appendix J of the Lewis book of pages 226-234 if the Evans and Flanagan book.)
  • There must be a comment directly above each method (including constructors) that is in proper JavaDoc format and states what task the method is doing, not how it is doing it. The comment should include tags for any parameters, return values and exceptions, and the tags should include appropriate comments that indicate the purpose of the inputs, the value returned, and the meaning of the exceptions.
  • There must be a comment directly above each field that, in one line, states what the field is storing.
  • There must be a comment either above or to the right of each non-field variable indicating what the variable is storing. Any comments placed to the right should be aligned so they start on the same column. Variables that are only used as an index in a loop do not need a comment.
  • There must be a comment above each loop that indicates the purpose of the loop. Ideally, the comment would consist of any preconditions (if they exist) and the subgoal for the loop iteration.
  • Any code that is complicated should have a short comment either above it or aligned to the right that explains the logic of the code.

The radix sort algorithm

The radix sort algorithm is the fastest way to alphabetize a list of words if the number of words is large. For this, you need an array of 27 linked lists. The linked lists will be for values 'a' through 'z' plus 'blank', and each list should start out empty. Create a loop that runs through the indeces. The loop should start at the largest index of the longest word and work down to 0. Let index be the current index you are considering in the loop. There should be an inner loop that runs through all the words in the linked list in order. If index character of the word is (upper or lower case) 'a', then place the word into the linked list for 'a'. If index character of the word is (upper or lower case) 'b', then place the word into the linked list for 'b'. And so on. If the word does not have a character at index (because the word is too short), then place it into the linked list for 'blank'. Once all the words are in their lists, append all the lists together starting with the blank list, then the 'a' list, then the 'b' list, and so on. Now, decrement the index and repeat the outer loop using the new list and resetting the array to again be 27 empty linke lists.

Here is an example. Suppose our list is initially: cat, bit, cop, cap, be, car, dart, bat, cart. Since 4 is the length of the longest word, we will start at 3 and move down to 0. We will place the words in order into the lists. After the first round, we have: 

"blank"  't'
------------
cat      dart
bit      cart
cop
cap
be
car
bat
(Note: I am only showing the array entries in which the list is not empty. For the other 25 array indeces not show, the array will hold an empty linked list.)

We append the lists in order, and the result is cat, bit, cop, cap, be, car, bat, cart, dart. Then we repeat on index 2 and run through the list in order placing the words into our array of lists. The array of lists is now: 

"blank" 'p'   'r'   't'
-----------------------
be      cop   car   cat
        cap   dart  bit
              cart  bat
We append the lists in order, and the result is be, cop, cap, car, dart, cart, cat, bit, bat. Then we repeat on index 1 and run through the list in order placing the words into our array of lists. The array of lists is now: 
'a'   'e'   'i'   'o'
---------------------
cap   be    bit   cop
car
dart
cart
cat
bat
We append the lists in order, and the result is cap, car, dart, cart, cat, bat, be, bit, cop. Then we repeat on index 0 and run through the list in order placing the words into our array of lists. The array of lists is now: 
'b'   'c'   'd'
---------------
bat   cap   dart
be    car
bit   cart
      cat
      cop
We append the lists in order, and the result is bat, be, bit, cap, car, cart, cat, cop, dart. Since we are done with all the indeces, the algorithm is finished, and the list is in sorted order.

Important change: Each time you go to add a word to the linked list, if the word is already at the end of the list, instead of adding it, change the count of the word to be the sum of the count of the word already on the list and the count of the word you will add. As a result, your list should never have consecutive words that are the same.
